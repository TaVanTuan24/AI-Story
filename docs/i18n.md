# Internationalization

AI Story supports two language concepts that are intentionally separate:

- Interface Language: the app UI, navigation, labels, buttons, toasts, loading states, and errors.
- Story Output Language: generated story prose, choices, summaries, titles, and recaps sent through the AI pipeline.

Supported languages for both are:

- `en` English
- `vi` Vietnamese

## UI Structure

Client i18n lives in:

```text
src/lib/i18n/types.ts
src/lib/i18n/dictionaries.ts
src/components/providers/i18n-provider.tsx
src/components/i18n/language-switcher.tsx
```

`I18nProvider` wraps the authenticated app in `AppProviders`. It exposes:

```ts
const { language, setLanguage, t, dictionary } = useI18n();
```

Translation keys are grouped by product surface:

- `common`
- `nav`
- `landing`
- `auth`
- `dashboard`
- `create`
- `play`
- `settings`

Use `t("section.key")` for short labels and `dictionary.section.arrayName` for translated arrays.

## Persistence

Anonymous users store interface language in local storage:

```text
ai-story.interface-language
```

Logged-in users persist language preferences in `UserPreference`:

```ts
{
  interfaceLanguage: "en" | "vi",
  storyOutputLanguage: "en" | "vi"
}
```

When a logged-in user changes interface language, the switcher updates local storage and patches `/api/me/preferences`.

## Settings UX

The visible language controls are in Profile Preferences:

- Interface Language uses the language switcher and changes app chrome immediately.
- Story Output Language uses a separate selector and affects future generated story text.

The AI Provider Settings language tab explains the distinction but does not merge language with model routing. Provider/model routing and story language remain independent.

## Story Output Language Pipeline

Story output language is read from user preferences when a session is created and stored in session metadata. When a session starts, it is copied into `StoryState.metadata.storyOutputLanguage`.

Context packs include:

```ts
language: {
  storyOutputLanguage: "en" | "vi",
  instruction: string
}
```

`StoryAiOrchestrator` appends a language instruction to every AI task request. The instruction tells the model to write player-facing generated text in the selected language while keeping JSON keys, enum values, ids, and schema fields stable.

This means a user can use the Vietnamese interface while generating English stories, or use the English interface while generating Vietnamese stories.

## What The User Sees

- The create-session page shows which story output language new sessions will use.
- The play page shows the active story output language for the current session.
- Prompt inputs and context packs carry that same language choice into generation, recap, and fallback output paths.

## Adding More UI Strings

Do not hardcode new product UI copy in components when it belongs to shared surfaces. Add keys to both `en` and `vi` dictionaries, then read them through `useI18n`.

For provider names, model names, IDs, logs, and API enum values, keep the canonical machine-readable value unchanged unless it is purely display copy.

## Encoding Rules

- Keep `src/lib/i18n/dictionaries.ts` saved as UTF-8.
